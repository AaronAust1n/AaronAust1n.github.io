# AI决策的真正瓶颈不是算法，而是你无法形式化业务逻辑

> 当你需要拿几百万赌一个决策时，你会发现你更想知道的不是"会怎样"，而是"这TM是怎么算出来的"。

*发布于 2026年3月2日*


我花了两天时间搭了一个完整的企业模拟系统——880个员工，66830笔交易，24个月的财务数据。目的是验证一件事：**本体论+AI能不能让决策真正可解释**。

结论前置：能。但不是你想象的那种"可解释"。

## 大家误解了本体论的作用

去年和几个做企业AI的朋友聊，他们都在用大模型做决策助手。我问怎么处理业务逻辑，回答都很相似："让模型学啊，训一训就行了。" 有人甚至觉得本体论是90年代的遗老技术，不如直接上Transformer。

这个想法的问题在：你把"学习模式"和"执行规则"混在一起了。

举个具体场景。我的模拟系统里有个问题：2025年Q1利润突然从77%的毛利率掉到72%。用纯机器学习的话，它会告诉你相关因素——原材料成本上涨8%，某品类销量下降15%，都对。但它不会告诉你**为什么**。

本体论干的事情是：它知道"毛利率"由什么构成，"原材料成本"从哪个供应商来，"供应商B"的"质量等级"是C而且"涨价15%"，而这个供应商服务的恰好是"IoT传感器"品类。然后顺着这条链往下推：成本上升→提价→需求下降→营销预算被挤压→销量进一步下滑。

这不是"发现相关性"，而是**追溯因果链**。前者靠统计，后者靠关系。

## 但本体论听起来很笨重

说实话，确实笨重。我定义了11444个实体，1915条关系，40多条规则。相比之下，直接喂给GPT一堆数据，让它自己琢磨，多省事。

问题是：当你需要调试的时候呢？

我在测试供应链决策时，系统给出了一个建议："对产品X下2000台订单"。如果这是个黑盒，你只能相信或不相信。但因为有本体，我可以看到完整的推理链：

```
当前库存2500台
日均需求84台（从交易记录计算）
供应商交付周期45天
订货点公式：(84×45)+1000=4780台
缺口：4780-2500=2280台
→ 建议订购2000台
```

每一步都能追溯到具体的数据表、具体的字段、具体的计算规则。这不是"AI觉得应该这样"，而是"根据这些事实和这条规则，逻辑上应该这样"。

区别在于：前者出错你不知道哪里错了，后者出错你知道是数据错了还是规则错了。

## 真正有意思的地方：规则和学习可以分层

我在这个系统里做了个实验：把推理分成三层。

**规则层**处理确定性逻辑——如果库存低于安全库存且交付周期大于7天，触发预警。这是死的，不需要学。

**图层**处理关系推理——从"利润下降"这个节点，往回追溯可能的因果路径。这用图算法，不是机器学习。

**统计层**处理不确定性——预测下个季度的需求，识别异常的成本波动。这才是ML的地盘。

三层配合的结果是：规则给出框架，统计填充细节。当统计层说"Q4销量可能增长40%"，规则层会检查：当前库存够不够？供应商产能如何？需要提前多久下单？然后给出一个不是"可能"而是"应该"的决策。

这种分层有个好处：**每一层都可以单独优化**。规则错了改规则，模型飘了重新训练，不会互相污染。纯端到端的黑盒做不到这点——你根本不知道是哪一层出了问题。

## 自我质疑：那为什么大公司不都这么干？

写到这里，我得承认一个问题：如果本体论这么好用，为什么大部分企业AI项目还是走纯数据驱动的路线？

答案可能让人不舒服：**建本体太贵**。

我这个模拟系统，为了定义"员工-部门-绩效-薪资-流失风险"这一条推理链，我写了200多行代码。真实企业的业务逻辑比这复杂十倍。你需要业务专家坐下来，把所有隐性知识显性化，这是个漫长而痛苦的过程。

相比之下，扔一堆数据给模型训练，三周就能上线一个demo。虽然不知道为什么work，但至少能跑起来。

所以本体论的使用场景可能更窄：**你需要可解释性超过便捷性的时候**。比如金融风控（需要向监管解释为什么拒贷），比如医疗诊断（需要告诉医生为什么给这个方案），比如关键供应链决策（需要向董事会证明为什么值得投这200万）。

## 一个更激进的猜想

但这里有个更有意思的可能性。

现在的LLM已经能理解自然语言描述的业务逻辑了。那能不能这样：用自然语言描述规则，让LLM把它转成形式化的本体定义，然后用这个本体去指导推理？

我测试了一下。给GPT描述："如果员工绩效大于4分但薪资低于同级别平均水平的90%，标记为流失风险。" 它能生成对应的规则代码。虽然不完美，但方向是通的。

这意味着：本体论的构建成本可能在未来几年断崖式下降。届时真正的瓶颈变成：**你能不能把业务逻辑说清楚**。

而这恰恰是很多企业的真问题——不是技术不行，是自己都没想清楚业务到底怎么运转的。

## 回到最开始那个问题

AI决策的瓶颈到底在哪？

我的答案是：在于你没有一个能让AI"看懂"业务逻辑的语言。统计模型只能看到相关性，看不到因果关系。规则系统能看到因果，但处理不了不确定性。

本体论+AI的价值在于：它提供了一个中间层，让业务逻辑和统计推断能够对话。

这不是说本体论是银弹。它笨重、昂贵、需要大量前期投入。但在那些真正需要"为什么"而不只是"是什么"的场景里，这可能是唯一可行的路径。

我把这套模拟系统开源了。880员工，15部门，66830笔交易，完整的推理引擎。如果你也想试试用本体论搭一个可解释的决策系统，可以拿去玩。

最后留一个问题：如果AI能完美预测但无法解释，和能解释但预测有偏差，你选哪个？

大部分时候，我们以为自己要的是准确率。但真到了要拿几百万赌一个决策的时候，你会发现你更想知道的是：**这TM是怎么算出来的**。
