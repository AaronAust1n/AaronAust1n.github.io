### [原创] 当软件正在变成手工艺，我们正在制造一场信息大萧条

GitHub上每天新增几十万个仓库。其中大多数没人用，没人维护，没人读。它们只是存在着，占据着服务器空间，在搜索结果里制造噪音，等待着被遗忘。说的是"开源生态繁荣"，说的是"贡献者数量创历史新高"，说的是"民主化"。但如果你打开任何一个热门话题下的 repository 列表，你看到的是：五十个功能几乎相同的 CLI 工具，二十个"我用周末写的 OpenClaw国产平替"，还有十几个 README 写得比代码长三倍的"框架"。这些代码不是在等人用。它们在等人收藏，然后再也不被打开。

我的判断是：**我们正在亲手制造一场信息的大萧条**。不是信息消失了，是信息多到让自己失效了。大量的开源软件从来就不是为了被使用而写的，而是为了被写而写。GitHub 不是代码仓库，是程序员的QQ空间（暴露年龄了）。

---

这件事是从一个很具体的转变开始的。

以前，如果你需要把一堆PDF合并、加水印、批量重命名，你会去找一个专门的工具。市场足够大，就会有人写一个App，放到GitHub上，慢慢积累用户。这是开源生态的正常运转：需求驱动供给，供给沉淀成工具，工具服务下一批有同样需求的人。

现在这条链断了。

用 Claude Code 造一个定制工具，开发时间从几天压缩到几小时，甚至十分钟搞定，完全符合你的具体情况，用完可以扔掉。你不需要那个GitHub上三年没更新、Issue区堆满bug报告的PDF工具了。这不只是效率的提升——这是需求的去中心化。长尾需求不再需要长尾软件来接住，因为每个人都可以按需生产自己的工具。不需要等维护者，不需要迁就别人的 API 设计。

你有一个很具体的需求——比如把你公司内部格式的日志文件解析成你能看懂的报表——以前你有三个选项：找一个凑合的开源工具改改，花时间自己写，或者忍着用 Excel。现在有第四个选项：十分钟，让 Claude Code 给你写一个刚好符合你需求的脚本，跑完就扔，或者扔进某个 `/tools` 文件夹再也不动。软件在回归手工艺。每个人造自己的工具，用完就放，不指望别人维护，也不指望别人用。

听起来是好事。问题在于：大量的开源项目，当初就是为了接住这些长尾需求而生的。

---

GitHub现在像一个巨大的跳蚤市场，里面九成的摊位卖的是同一款劣质商品。`awesome-xxx` 清单套清单，tutorial套tutorial，用Claude生成的README比原来的代码还长。这不是开发者素质下降的问题，是激励结构坏掉了：Star数量可以用bot刷，被引用一次就能写进简历，低成本发布带来的沉没成本极低，所以没人在意质量。

结果是，真正有价值的库淹没在垃圾里。你搜一个功能，前五条结果有四条是AI生成的仿制品，剩下一条是真的，但文档写得一团糟。信息的数量在增长，检索的成本也在同步上涨——而且上涨得更快。

这是信息通货膨胀的核心机制：**生产信息的边际成本趋近于零，但消费信息的成本没有跟着下降**。你反而要花更多时间辨别真假，过滤噪音，确认那个看起来能用的包里有没有藏着恶意代码。

---

有人会说：搜索引擎会解决这个问题，AI摘要会解决这个问题，社区的筛选机制会解决这个问题。

这是一个过于乐观的信念。

搜索引擎的排名算法已经在和SEO垃圾打了二十年的持久战，现在又多了一个对手——AI生成内容，产量是人工的几百倍。Stack Overflow上的答案开始出现AI生成的错误解答，而且得票数不低，因为看起来很像正确答案。Wikipedia的编辑在和自动生成的词条争地盘。学术数据库里开始混入用LLM生成、仅做了最低限度修改的论文。

筛选机制的速度永远比生产机制慢一拍。

---

如果每个人都在造自己的工具，那么 GitHub 的"开源垃圾"问题会更严重，还是会消失？答案可能是：都不会。会发生的是**分层**。

一层是真正的基础设施——少数几个被大量使用的框架、语言工具链、数据库。这一层会更稳固，因为它们的价值不在于长尾需求，而在于普遍性。Python 不会消失，但那五十个"Python 写的 todo app"会越来越没有意义。

另一层是私有的、一次性的工具——你用 Claude Code 造的那个脚本，你公司内部用的那个小工具，它们可能根本就不会上 GitHub，或者上了也没有人需要找到它。这一层会膨胀，但不会堆在公共空间里造成噪声。

真正的问题是中间那层：那些雄心勃勃但执行平庸的开源项目，那些 star 数到了几百然后就再也没有 commit 的仓库。这一层不会因为 Claude Code 而消失，反而可能因为"任何人都能生成代码"而变得更拥挤。门槛降低了，但辨别信号的成本没有降低。

把这个趋势推到极端，我想问：信息增殖、内容贬值、重复建设——互联网会不会就此消亡？

---

所以，互联网会消亡吗？

这个问题本身可能问歪了。互联网不会消亡，但互联网作为一个**可信信息基础设施**的功能正在被侵蚀。它还会在，只是你越来越难相信你在上面读到的东西。它也可能会分裂成两种不同的使用方式。一种是通过 AI 界面访问的互联网——你不需要知道底下运行的是什么代码，你描述需求，得到结果，就像你打开水龙头不需要知道管道怎么走。另一种是直接访问原始层的互联网——文档、源码、数据——但这一层的使用者会越来越少，越来越专业。

这两种使用方式之间的裂缝正在扩大。普通用户越来越少接触"互联网的原材料"，越来越多接触的是被 AI 处理过的结果。这不是消亡，这是一次旧工业和新工业的层叠：钢铁没有消失，但大多数人不再需要知道如何炼钢。

也许这个问题更准确的问法也许是：**当信息不再可信，网络连接本身还有多少价值？**

我们现在建的那些用完即扔的Claude Code脚本，至少是真实的——它解决了你的问题，你亲眼看到它跑通了。这是一种奇怪的退回：从依赖公共信息基础设施，退回到依赖自己能验证的东西。社区的、共享的、可积累的知识——这些互联网最初许诺要带来的东西——正在变得难以维系。

这不是互联网的消亡。这是互联网许诺的那个未来，在它自己手里变质了。

---

我本人遭遇的案例：我发现WPS提供了插件开发功能，找了几个开源项目发现没法运行，于是用cursor自己写了一个，差不多两天就写完了，然后提交到了公司的插件市场，插件原型也开源了。结果当然是无人问津，因为根本没人知道有这个插件。
