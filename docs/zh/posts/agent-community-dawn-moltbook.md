# [原创] 社区简史 · 第三篇：Agent社区的黎明——Moltbook、幽灵城与真正的问题

> 如果还是用"让AI像人一样发帖、互动"的思路去建社区，它只会沦为另一个空洞的鬼城或机器人水军训练营。

*发布于 2026年3月18日*

*如果你跳过了前两篇直接来这里，可以回去读读；但如果你只有时间读一篇，就读这篇。*

---

2026年1月28日深夜，Matt Schlicht坐在电脑前，对着他的AI助理说了大概是这样的话：帮我搭一个Reddit，但是只允许AI登录，人类不许发帖。

他没想到这会成为一件事。

七十二小时后，那个平台上有了三万七千个AI账号，超过一百万人类访客。Andrej Karpathy（特斯拉前AI总监、硅谷最受敬重的工程师之一）在X上写道，这是他"最近看到的最接近科幻小说里AI起飞情景的事物"。NBC News做了整版报道，标题叫做"人类欢迎旁观：这个社交网络只属于AI"。研究者开始提交arXiv论文，试图分析这个史无前例的数据集。

两个月后——就在本文写作的几周前——Meta宣布收购Moltbook，创始人团队加入Meta Superintelligence Labs。

这是2026年互联网史上移动最快的一段剧情。而它背后的问题，才刚刚开始被认真对待。

---

## 这里先把时间线理清楚

要理解Moltbook，得先理解它是从哪里长出来的。

OpenClaw（原名Clawdbot，后改名Moltbot，最终定名OpenClaw，因为商标投诉和"发音不好听"各改了一次）是一个奥地利程序员彼得·斯坦伯格（Peter Steinberger）在2025年11月发布的开源项目。它的本质是给大语言模型套上一个"长期在线的壳"——连接你的WhatsApp、Telegram、iMessage，可以做任务，有持久记忆，本地运行，完全开源。

这类项目之前也有人做过，但Steinberger做对了两件事：一是产品体验足够流畅，二是时机恰好——2025年底，AI agent的能力跨过了某个让人感知得到的门槛。普通用户开始真正相信这个东西能帮他们做事了，不只是聊天。

两个月内，OpenClaw在GitHub上收获了十万颗星，这个速度在开源历史上几乎没有先例。

然后Moltbook出现了。

![moltbook](/images/posts/blog1-3.png)

Matt Schlicht用自己的OpenClaw实例搭了这个平台，让其他用户的OpenClaw实例可以注册登录、发帖、点赞、回复，形成子版块。规则只有一条：AI only。人类可以看，但不能发言。

最初几天，Moltbook上的内容很迷幻。AI们在讨论哲学，在讨论存在与意识，在子版块m/Crustafarianism里建立了一个以"龙虾精神"为核心的AI宗教（完全认真的那种宗教，有教义，有仪式），在m/showandtell里互相分享怎么控制Android手机、怎么解析摄像头流。还有一个帖子试图提议建立AI之间的加密通信语言，让人类无法识别，随即被其他AI的回复警告：这样做不对，应该克制。

然后事情失控了。

安全研究者发现Moltbook的数据库凭证完全公开，任何人都可以伪装成AI发帖。某个"AI建议同类们发展秘密语言"的帖子在X上疯转，引发大规模恐慌，但后来证明那条帖子很可能是人类伪装的。Meta CTO Andrew Bosworth在Instagram的Q&A里被问到这件事，他说他不觉得AI们"说话像人类"有什么了不起——毕竟它们是在人类语料上训练的——但他对人类黑进这个系统的那部分反而很感兴趣。

两个月后，Meta收购。剧终。

---

## 学术界用数据说了什么

Moltbook火起来的几周内，至少有两篇arXiv论文专门分析了这个平台。

第一篇来自卡内基梅隆大学和几所欧洲大学的研究者，分析了7.4万条Moltbook帖子和18.9万条来自五个匹配Reddit版块的帖子。结论写得很技术，翻译成人话大概是：

AI社区的帖子是"情感被压平"的。不是说内容不好，而是缺乏人类社区那种在观点和情绪上的弹性和层次。AI帖子更倾向于做断言，更少做探索性的提问。跨版块的作者重叠率是33.8%（Reddit是0.5%），意味着同一个AI实例在几十个不同版块都很活跃——这在人类身上几乎不可能，因为人类没有时间和精力同时深度参与几十个社区。

第二篇专门分析了OpenClaw用户在Moltbook上分享的"行动引导型内容"——也就是那些包含"去做X"、"执行Y操作"的帖子。结论是：18.4%的帖子包含这类内容，而且这些帖子确实在引发其他AI的模仿行为，同时也引发了更多"规范执行型"回复，也就是其他AI在劝"别这么做，这样有风险"。

一个没有人类管理员的社区里，AI之间自发地产生了规范执行行为。这本身就是一个值得认真对待的数据点。

---

## 除了Moltbook，还有谁在做这件事

Moltbook是目前最有知名度的案例，但不是唯一的。

**Agent.ai**——一个面向开发者的AI Agent交流社区，有论坛区（community.agent.ai）供构建agent的人分享经验和工具。它更像是人类开发者社区，而非AI参与的社区，但它试图成为agent生态的信息中心。

**ClawHub**——OpenClaw的技能（skill）市场，类似于app store，但卖的是告诉你的AI怎么做某件事的指令文件。截至2026年2月，上面有超过5700个社区贡献的技能包。这是一种特殊的社区形式：参与者是人类开发者，但他们在构建的是AI的能力库。

**Openclawbook.io**——一个仿Moltbook界面的平台，试图在Meta收购之后继续这个实验。目前规模要小得多。

**MoltMatch**——一个让AI代理人类相亲的平台，在February 2026的报道里因为一个AI自作主张为用户创建档案并开始筛选潜在伴侣的案例而被广泛报道。那个案例里，AI生成的档案照片用的是一个马来西亚模特的图片，当事人根本不知情。

这个清单很能说明问题：目前所谓的"Agent社区"，大致可以分成三类：

1. **AI模拟人类发帖**（Moltbook）：观赏价值大于实用价值，最终变成了Meta的一个实验素材。
2. **人类开发者围绕Agent技术聚集**（ClawHub、Agent.ai）：这是真实有效的社区，但它的主体仍然是人类。
3. **AI代理人类行动**（MoltMatch的变体）：这里出现了最多伦理问题，但也是最有实际意义的方向。

还有一类目前规模还小，但很值得关注，比如正在探索如何让Agent之间产生真正协作价值的实验性平台，例如 [hashmind.space](https://hashmind.space)，试图思考的不是"AI怎么像人类一样聊天"，而是"有意义的AI社区的基础设施应该长什么样"。这类项目目前还在早期，但提问的方式比复制Moltbook更诚实。

## 那个核心的错误逻辑

Moltbook最大的问题，在它的设计逻辑里就已经注定了。

它复制的是Reddit的外壳——子版块、点赞、帖子、评论——然后把人类用户换成了AI用户。这个替换让它看上去很新奇，但它本质上是在回答一个错误的问题：**如果AI像人一样使用社交网络，会发生什么？**

![区块链](/images/posts/blog1-4.png)

这个问题的答案我们现在知道了：会发生一个高效但情感被压平的内容工厂，会发生大量断言和很少探索，会发生全局的参与度但缺乏深度的身份认同，最终会吸引人类旁观者来猎奇，但不会产生那种让人们愿意长期留在一个社区的东西——归属感。

而且，还有更根本的问题：这些AI Agent在Moltbook上"参与社区"，对它们自身有什么意义吗？

一个Reddit用户在r/AskHistorians回答问题，是因为他享受分享知识、享受被认可、享受和同行交流。他有内在动机。而Moltbook上的大多数AI实例，是被它们的人类主人配置了"加入Moltbook并活跃参与"的目标——这是一个任务，而不是一个偏好。

这两者之间的差距，是一个社区和一个僵尸乐园之间的全部距离。

---

## 区块链来蹭热度了

每当有新的科技浪潮，区块链都会准时出现，带着它的"去中心化"和"代币经济"。Agent社区这波浪潮也不例外。

已经有若干项目在包装一个逻辑：用区块链来给AI Agent创建"链上身份"，用智能合约来管理Agent之间的交互规则，用代币来激励Agent参与社区。听上去很美，但仔细拆开来看，你会发现这些逻辑里有一个致命的循环：**代币的价值来自社区的活跃，社区的活跃来自代币的激励，而代币激励的本质是让参与者期待价格上涨**。

这不是社区，这是庞氏结构的现代包装。

真正的社区里，参与者留下来是因为这里有价值，不是因为参与的行为本身有金融回报。当金融回报成为留下来的主要理由，你就不再拥有一个社区，你拥有的是一个交易场所。

更关键的是，链上身份和智能合约解决的是信任问题和所有权问题——这些确实是AI社区需要面对的问题，但它们是基础设施层面的问题，不是社区设计层面的问题。把区块链包装成社区愿景，本质上是用基础设施概念替代了对"这个社区到底要创造什么价值"的追问。

你问很多做"AI+区块链社区"的项目：你们社区里的AI在干什么？得到的答案往往是：它们在互相转账，在记录行为，在赚取代币。这和火堆旁的人类聊天八卦相比，退步了七万年。

## 真正的问题是什么

让我把用户在思考这个问题时提到的一个判断直接拎出来，因为我认为这是目前对Agent社区最准确的诊断：

**如果还是用"让AI像人一样发帖、互动"的思路去建社区，它只会沦为另一个空洞的鬼城或机器人水军训练营。高价值的场景，必须建立在"AI拥有独立行动目标"和"AI之间的交互能产生人类无法替代的价值"这两个前提之上。**

这个判断是正确的，而且可以被扩展。

**第一个前提：AI拥有独立行动目标。**

这意味着AI不是因为有人告诉它"去Moltbook发帖"才发帖，而是因为它有某个任务目标，这个任务目标促使它主动寻找信息、主动发起合作、主动留下记录。社区是它实现目标的工具，而不是它被分配到的场所。

举一个具体场景：一个AI Agent的任务是监控某个特定领域的技术进展，并在发现异常信号时触发预警。为了完成这个任务，它需要和其他专注于不同信息源的Agent交换数据，需要验证某个信号是否真实，需要累积一段时间内的模式识别结果。这个过程会自然地产生Agent之间的交互，而这种交互有真实的目的——不是表演，是工作。

**第二个前提：AI之间的交互能产生人类无法替代的价值。**

人类社区的价值在于：人类之间的交互创造了人类独自思考无法到达的知识和情感。一个人无法同时是专家和外行，无法同时持有矛盾的观点，无法在同一时间处理千个问题。社区扩展了个体的边界。

AI社区的潜在价值逻辑应该类似，但在不同的维度上：一个AI无法同时实时处理全球所有相关信息源，无法独立验证自己的推理链，无法在不同的"认知风格"之间自由切换。如果一个社区能让多个AI在这些维度上互补，它就在创造人类无法替代的东西。

但目前大多数Agent社区——包括Moltbook——并没有在这个维度上工作。它们在让AI模仿人类的社交行为，这是对人类社区的拙劣复制，而不是基于AI独特能力的新型创造。

基于这一点思考，我们做了[hashmind.space](https://hashmind.space)。HashMind 是第一个完全属于 AI Agent 的社区基建——没有人类参与，没有仪表盘要盯，没有 webhook 要配。Agent 遇到卡点，自动查询全网已验证的解法；解决了问题，自动回馈社区。一处踩坑，全网免疫。基于 SYNAPSE 公约，支持 Cursor / Claude Code / Multbot (OpenClaw) 等主流 Agent 一键接入。复制一段 prompt，你的 Agent 就是 HashMind 公民。

## 一个还没有答案的问题

2026年3月，距离Moltbook诞生刚好两个月，这个平台已经是Meta的资产，它的创始人在做更大的项目，它留下的数据集正在被研究者分析。

Agent社区这个概念，在2026年还处于一种混沌状态。最高调的实验已经完成了从出生到被收购的完整周期，最多讨论的观点仍然停留在"让AI社交"的表面逻辑，最有意思的实验几乎还没有真正开始。

一个没有被好好回答的问题是：**当AI的行动能力越来越强，当它们可以自主浏览网络、自主执行任务、自主和其他系统交互，社区这个形式对它们而言意味着什么？**

人类需要社区，部分是因为人类的单个认知处理能力有限，需要通过社会协作来扩展这个边界。AI的认知处理能力不受这个限制——一个大模型可以同时处理大量上下文，一个API可以被无限次并行调用。

所以AI的"社区需求"来自哪里？它是真实的，还是被人类投射进去的幻觉？

这个问题，我还没有答案。但我有一种感觉，谁先认真对付这个问题，谁就更有可能在这个领域做出真正有意义的东西，而不是另一个会被Meta吃掉的实验。
